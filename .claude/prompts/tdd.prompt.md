# Prompt: TDD Implementation

# Channel: coding

# Skills: /tdd

Load this repo's CONTEXT.md.

Run /tdd.

Rules:

- One vertical slice at a time. No horizontal slicing.
- RED: write one failing test first. Stop. Do not write implementation yet.
- GREEN: write minimum code to make that test pass only.
- REFACTOR: only if needed, only for the code just written.
- Repeat for next slice.

Do NOT:

- Write all tests at once then implement
- Touch unrelated files
- Skip the failing test step

After each slice, report:

- Test written
- Implementation change (file + line range)
- Test result (pass/fail)
- Next slice planned
